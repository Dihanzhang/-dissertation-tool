### §7.7 — Copyright Attribution for Reprinted Tables and Figures [MISSED]

**What was missed:** When reprinting or adapting a table or figure, include a copyright attribution in the note. Use "Reprinted from" for verbatim reproductions and "Adapted from" for modified versions. Copyright attribution format:

### §7.8 — Accessibility of Tables and Figures [MISSED]

**What was missed:** Tables and figures should be accessible. Provide alternative text (alt text) descriptions for figures in digital publications. ### §7.9 — Table Components [MISSED]

**What was missed:** A standard table has five components: (1) table number, (2) title, (3) headings, (4) body, (5) notes. The stub column (leftmost column listing row names) is required. **Rule:** All five components are standard. Every column — including the stub — requires a heading.

### §7.11 — Table Titles [MISSED]

**What was missed:** Each table must have a brief, explanatory title. Avoid overly general titles ("Summary of Results") and overly detailed titles that replicate the note. The title appears in italic title case, one double-spaced line below the table number, flush left.

### §7.12 — Table Headings [MISSED]

**What was missed:** All column headings, including the stub column heading, appear in sentence case (not title case). Headings are centered above their columns. ### §7.13 — Table Body [MISSED]

**What was missed:** Use an em dash (—) to indicate a cell for which data are not available (the researcher looked for data but could not find it). Leave a cell blank when the datum is not applicable (the intersection does not logically exist). Do not use "N/A" or "n/a" unless defined. Do not use ellipses to fill empty cells. **Rule:** Em dash = data not available. Blank = not applicable. Consistent decimal places within a column.

**Checkable:** YES

**Confidence:** HIGH

### §7.14 — Table Notes [MISSED]

**What was missed:** Tables may have three types of notes, each beginning on a new line below the table:

1. **General note:** begins with "*Note.*" in italic, followed by a period. Applies to the table as a whole. Defines abbreviations, gives copyright attribution, explains special formatting.
2. **Specific note:** marked by superscript lowercase letters (a, b, c), assigned left-to-right, top-to-bottom across the table. Begin the specific note with the superscript letter: "a n = 50."
3. **Probability note:** marks statistical significance using asterisks. Standard convention: *p* < .05. **p* < .01. ***p* < .001. Must be consistent within the same paper (same asterisk = same p threshold throughout).

Order of notes: general note first, then specific notes, then probability notes. Only include the note types that apply.

**Rule:** Three types of notes in fixed order. Asterisk/p-value convention must be consistent across all tables in a paper.

**Checkable:** YES

**Detection:** Flag inconsistent asterisk thresholds across tables in same paper. Flag specific notes not assigned sequentially left-to-right, top-to-bottom. Flag general notes not beginning with "*Note.*" in italic.

**Confidence:** HIGH

### §7.15 — Standard Abbreviations in Tables and Figures [MISSED]

**What was missed:** Standard abbreviations recognized in the field do not need to be defined in a table note (e.g., M, SD, SE, CI, df, F, t, p, n, N, %, ms). Non-standard abbreviations (author-created or uncommon) must be defined in a note. Statistics, Greek letters, and units of measurement do not require definition in the note.

**Rule:** Common statistical abbreviations need no definition. Author-specific abbreviations must be defined.

**Checkable:** YES

**Detection:** Flag table notes defining standard abbreviations (M, SD, etc.) unnecessarily; flag tables using non-standard abbreviations without defining them.

**Confidence:** HIGH

### §7.16 — Confidence Intervals in Tables [MISSED]

**What was missed:** When reporting confidence intervals alongside point estimates in tables, present the CI in square brackets immediately after the point estimate, or place CI limits in separate lower-limit (LL) and upper-limit (UL) columns. State the CI level (e.g., 95%) in the column heading or table note. Do not use "CI" without specifying the level.

**Rule:** CI must specify the level (95%, 90%, etc.). Present in brackets or separate LL/UL columns.

**Checkable:** YES

**Detection:** Flag "CI" in table headings without a percentage (e.g., "95% CI" is correct; "CI" alone is insufficient).

**Confidence:** HIGH
